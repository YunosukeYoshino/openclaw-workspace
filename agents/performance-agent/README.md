# Performance Agent

パフォーマンス管理エージェント - 最適化とベンチマーク追跡

## 概要

Performance Agentは、システムパフォーマンスの監視、ベンチマーク、最適化を管理するためのエージェントです。

## 機能

### メトリクス管理
- パフォーマンスメトリクスの記録と追跡
- トレンド分析
- コンポーネント別・環境別フィルタリング

### ベンチマーク
- ベンチマークの作成と追跡
- ベースラインと目標値の設定
- 実行結果の記録

### 最適化管理
- 最適化計画の追加
- 改善率の自動計算
- 進捗状況の追跡

### アラート
- パフォーマンス異常の検知
- 重要度レベルの設定
- 解決状況の追跡

### レポート
- パフォーマンスサマリー
- 定期レポートの生成
- インサイトの記録

## 使い方

### メトリクス追加
```
メトリクス追加 メトリクス:response_time 値:150 単位:ms コンポーネント:API
```

### ベンチマーク作成
```
ベンチマーク追加 名前:API Response タイプ:response_time 目標:100
```

### 最適化計画
```
最適化追加 名前:DBインデックス 前:500 後:200 単位:ms
```

### サマリー表示
```
サマリー表示
```

## テーブル構造

### metrics
- メトリクス名、値、単位、コンポーネント、環境、タグ、メモ

### benchmarks
- ベンチマーク名、タイプ、ベースライン、目標値、現在値、ステータス

### optimizations
- 最適化名、コンポーネント、説明、前後の値、改善率、ステータス

### alerts
- アラートタイプ、重要度、メトリクス、しきい値、メッセージ、解決状態

### reports
- レポート名、タイプ、期間、サマリー、インサイト

## データベース

SQLiteベースで、`performance.db`ファイルにデータが保存されます。
