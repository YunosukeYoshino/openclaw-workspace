# Scale Agent

スケール管理エージェント - スケーリングとキャパシティプランニング

## 概要

Scale Agentは、リソースのスケーリング、キャパシティプランニング、自動スケーリングを管理するためのエージェントです。

## 機能

### リソース管理
- リソースの追加と追跡
- 使用量の更新と監視
- 利用率の計算
- 環境別管理（production, staging, development）

### スケーリングイベント
- スケールアップ/ダウンイベントの記録
- イベントの追跡と完了管理
- リソースのスケーリング履歴

### 使用量履歴
- 使用量の履歴記録
- トレンド分析
- パフォーマンス推移の可視化

### キャパシティプラン
- 将来のキャパシティ計画
- 予測成長の記録
- 推奨容量の提案
- 優先度管理

### しきい値と自動スケール
- スケールアップ/ダウンしきい値の設定
- 最小/最大容量の制限
- 自動スケーリングの有効化
- 自動スケーリングトリガーの確認

## 使い方

### リソース追加
```
リソース追加 タイプ:CPU 名前:server-1 容量:100
```

### 使用量更新
```
使用量更新 ID:1 値:50
```

### スケーリングイベント追加
```
スケーリング追加 ID:1 スケールアップ 前:100 後:150
```

### しきい値設定
```
しきい値設定 ID:1 up:80% down:30% 最小:50 最大:200 自動:オン
```

### スケーリング確認
```
スケーリング確認
```

## テーブル構造

### resources
- リソースタイプ、名前、容量、現在の使用量、利用率、ステータス、環境

### scaling_events
- リソースID、イベントタイプ（scale_up/scale_down）、変更前後の容量、理由、ステータス

### capacity_plans
- プラン名、リソースタイプ、予測期間、予測成長率、推奨容量、予定日、優先度、ステータス

### usage_history
- リソースID、使用量、タイムスタンプ、メモ

### thresholds
- リソースID、スケールアップしきい値、スケールダウンしきい値、最小/最大容量、自動スケール有効フラグ

## データベース

SQLiteベースで、`scale.db`ファイルにデータが保存されます。
