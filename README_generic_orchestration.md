## 🚀 開発計画 (Development Plan)

### 基本方針 (Basic Strategy)

**自律開発 (Autonomous Development)**:
- ユーザーからの指示を待たず、自律的に開発を進める
- 各エージェントは `db.py` と `discord.py` の2ファイルで構成
- 日本語と英語の両方に対応
- テストコードを含める

**並列開発 (Parallel Development)**:
- 複数のエージェントを同時に開発可能
- バッチごとに5個ずつ開発
- 失敗したら再試行

### 開発フロー (Development Flow)

1. **エージェント選択**
   - 次のバッチ（5個）を自動選択
   - 例: 61-65, 66-70, 71-75...

2. **db.py作成**
   - SQLiteベースのデータベース管理
   - 必要なテーブル定義
   - インデックス、トリガーの設定

3. **discord.py作成**
   - 自然言語解析でメッセージを処理
   - 追加・更新・削除・検索・一覧・統計機能
   - テストコード

4. **検証**
   - 各ファイルを実行して動作確認
   - エラーがあれば修正

5. **記録**
   - MEMORY.md に進捗を追記
   - memory/YYYY-MM-DD.md に今日の進捗を追記

6. **次のバッチへ**
   - 5個完了したら、次の5個へ進む

### スケジュール (Schedule)

**目標**: 残り40個を可能な限り早く完了する

**推定時間**:
- 1つのエージェント: 約3-5分
- 1つのバッチ（5個）: 約15-25分
- 残り40個: 約8バッチ = 約2-3時間

## 🔧 問題解決手順 (Troubleshooting)

### よくある問題と解決策

**1. ファイル作成に失敗した**
   - 理由: ディスク容量、権限、パスの問題
   - 解決: 再試行する。それでもダメなら、エラーメッセージを記録して次へ進む

**2. コードの実行エラー**
   - 理由: シンタックスエラー、ライブラリの問題
   - 解決: コードを確認して修正する

**3. メモリの書き込みエラー**
   - 理由: データベースのロック、ファイルの競合
   - 解決: 数秒待ってから再試行する

**4. 進捗が遅い**
   - 理由: 複雑なエージェント、エラーによるリトライ
   - 解決: 単純なエージェントから先に作る
